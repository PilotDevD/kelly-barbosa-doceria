# 🎂 Kelly Barbosa — WebApp da Doceria

Guia **passo a passo** para colocar o site no ar. Não precisa saber programar!

Você vai precisar de 3 coisas (todas gratuitas):

1. Uma conta no **Supabase** (banco de dados)
2. Uma conta no **Vercel** (hospedagem do site)
3. Uma conta no **GitHub** (onde o Vercel busca o código)

Tempo estimado: **30 a 45 minutos** na primeira vez.

---

## Parte 1 — Criar o banco de dados (Supabase)

### 1.1 Criar a conta

1. Acesse **https://supabase.com**
2. Clique em **"Start your project"** (canto superior direito)
3. Entre com sua conta do **GitHub** (se ainda não tiver, crie uma grátis em https://github.com)

### 1.2 Criar o projeto

1. Clique em **"New project"**
2. Preencha:
   - **Name:** `kelly-barbosa`
   - **Database Password:** crie uma senha forte e **anote em papel** (você vai precisar de novo)
   - **Region:** `South America (São Paulo)`
   - **Plan:** Free
3. Clique em **"Create new project"** e aguarde uns 2 minutos.

### 1.3 Criar as tabelas

1. No menu lateral esquerdo, clique no ícone **"SQL Editor"** (parece um `>_`).
2. Clique em **"New query"**.
3. Cole **exatamente** o código abaixo e clique em **"Run"** (botão verde no canto inferior direito):

```sql
-- Tabela de produtos (cardápio)
create table produtos (
  id bigint generated by default as identity primary key,
  nome text not null,
  descricao text,
  categoria text default 'Outros',
  preco numeric(10,2) not null,
  foto_url text,
  ativo boolean default true,
  created_at timestamptz default now()
);

-- Tabela de pedidos
create table pedidos (
  id bigint generated by default as identity primary key,
  cliente_nome text not null,
  cliente_telefone text,
  cliente_endereco text,
  forma_pagamento text,
  observacoes text,
  itens jsonb not null,
  total numeric(10,2) not null,
  status text default 'novo',
  created_at timestamptz default now()
);

-- Permissões (Row Level Security)
alter table produtos enable row level security;
alter table pedidos enable row level security;

-- Qualquer pessoa pode LER produtos (cardápio público)
create policy "ler produtos"       on produtos for select using (true);
-- Só a Kelly (logada) pode MEXER nos produtos
create policy "gerenciar produtos" on produtos for all    using (auth.role() = 'authenticated') with check (auth.role() = 'authenticated');

-- Qualquer pessoa pode CRIAR pedidos (cliente finalizando compra)
create policy "criar pedidos"      on pedidos  for insert with check (true);
-- Só a Kelly (logada) pode ver/editar/excluir pedidos
create policy "ler pedidos"        on pedidos  for select using (auth.role() = 'authenticated');
create policy "atualizar pedidos"  on pedidos  for update using (auth.role() = 'authenticated');
create policy "excluir pedidos"    on pedidos  for delete using (auth.role() = 'authenticated');

-- Tempo real para a Kelly ver os pedidos chegando
alter publication supabase_realtime add table pedidos;
```

Se aparecer "Success. No rows returned" — perfeito! ✅

### 1.4 Criar o local de fotos (Storage)

1. No menu lateral, clique em **"Storage"**.
2. Clique em **"New bucket"**.
3. Name: `fotos` — ative **"Public bucket"** — clique **"Save"**.
4. **Volte para o "SQL Editor"** (ícone `>_`, mesmo lugar onde você rodou o primeiro SQL). Clique em **"New query"**, cole o SQL abaixo e clique **Run**:

```sql
create policy "leitura publica fotos"
  on storage.objects for select
  using (bucket_id = 'fotos');

create policy "upload fotos autenticado"
  on storage.objects for insert
  with check (bucket_id = 'fotos' and auth.role() = 'authenticated');

create policy "atualizar fotos autenticado"
  on storage.objects for update
  using (bucket_id = 'fotos' and auth.role() = 'authenticated');

create policy "deletar fotos autenticado"
  on storage.objects for delete
  using (bucket_id = 'fotos' and auth.role() = 'authenticated');
```

> ⚠️ **Importante:** esse SQL vai no **SQL Editor**, não na telinha de "New policy" do Storage (lá ele não aceita o `create policy`).

### 1.5 Criar o usuário (login da Kelly)

1. No menu lateral, clique em **"Authentication"** → **"Users"**.
2. Clique em **"Add user"** → **"Create new user"**.
3. Preencha:
   - **Email:** seu e-mail (ex: `kelly@gmail.com`)
   - **Password:** crie uma senha forte — **esta é a senha do painel!** Anote.
   - Marque **"Auto Confirm User"**.
4. **"Create user"**.

### 1.6 Copiar URL e chave do projeto

1. No menu lateral, clique no ícone de engrenagem **"Project Settings"** → **"API"**.
2. Você vai ver duas informações importantes:
   - **Project URL** (algo como `https://abcxyz.supabase.co`)
   - **anon / public key** (um texto enorme começando com `eyJ...`)
3. **Abra o arquivo `supabase.js`** no seu computador e substitua:

```javascript
const SUPABASE_URL = 'COLE_AQUI_SUA_URL_DO_SUPABASE';
const SUPABASE_ANON_KEY = 'COLE_AQUI_SUA_CHAVE_ANON';
```

Pela URL e chave que você copiou. Salve o arquivo.

⚠️ **A chave `anon` é pública e pode ser usada no site sem problema** — ela é limitada pelas regras de segurança que criamos acima. Nunca use a chave `service_role` no site!

## Parte 2 — Colocar o site no ar (Vercel)

### 2.1 Subir o código para o GitHub

1. Acesse **https://github.com** e entre com sua conta.
2. Clique no **"+"** no canto superior direito → **"New repository"**.
3. Repository name: `kelly-barbosa-doceria` → **Public** → **"Create repository"**.
4. Na próxima tela, clique em **"uploading an existing file"**.
5. Arraste **todos os arquivos** da pasta `kelly-barbosa-doceria` (index.html, admin.html, style.css, app.js, admin.js, supabase.js, README.md) para a janela.
6. Clique em **"Commit changes"**.

### 2.2 Hospedar na Vercel

1. Acesse **https://vercel.com** e clique em **"Sign Up"** (use sua conta do GitHub).
2. No painel da Vercel, clique em **"Add New..."** → **"Project"**.
3. Encontre `kelly-barbosa-doceria` na lista e clique em **"Import"**.
4. Deixe tudo no padrão e clique em **"Deploy"**.
5. Aguarde uns 30 segundos — seu site está no ar! 🎉

A Vercel vai te dar um endereço como `kelly-barbosa-doceria.vercel.app`.

- **Site do cliente:** `https://kelly-barbosa-doceria.vercel.app/`
- **Painel da Kelly:** `https://kelly-barbosa-doceria.vercel.app/admin.html`

### 2.3 (Opcional) Usar seu próprio domínio

Se você comprar um domínio (ex: `kellybarbosa.com.br`), na Vercel clique em **Settings → Domains** e adicione. Eles te guiam passo a passo.

## Parte 3 — Usando no dia a dia

### Para a Kelly (dona)

1. Abra `https://seu-site.vercel.app/admin.html` no celular.
2. Entre com o e-mail e senha que você criou no passo 1.5.
3. Na aba **Produtos**, cadastre seus bolos, doces e itens de papelaria (com foto, preço, descrição).
4. Na aba **Pedidos**, os pedidos chegam em tempo real — toque no cliente para abrir o WhatsApp dele!
5. Troque o status conforme for produzindo: `Preparando` → `Saiu para entrega` → `Entregue`.

### Para os clientes

É só compartilhar o link `https://seu-site.vercel.app/` — eles abrem, escolhem os doces, preenchem o endereço e finalizam. Você recebe no painel na hora.

## Como atualizar o site depois?

1. Edite o arquivo que quer mudar.
2. Vá no seu repositório do GitHub → clique no arquivo → ícone de lápis → edite → "Commit changes".
3. A Vercel atualiza o site automaticamente em segundos. ✨

## Personalização rápida

- **Mudar o WhatsApp:** no arquivo `index.html`, procure por `5522999999999` e `(22) 99999-9999` e troque pelos seus.
- **Mudar as cores:** no `style.css`, no topo do arquivo tem a seção `:root` com todas as cores (`--rose-500`, `--teal-500`, etc.). Trocando ali, muda tudo.
- **Mudar o nome/tagline:** direto no `index.html` e `admin.html`.

## Problemas comuns

| Problema | Solução |
|---|---|
| "Cardápio vazio" | Cadastre produtos no painel (admin.html) e marque "Disponível no cardápio" |
| "Não consigo logar no painel" | Confira se criou o usuário no passo 1.5 e se o "Auto Confirm User" estava marcado |
| "Pedido não salva" | Verifique se colou URL e chave corretas no `supabase.js` |
| "Foto não aparece" | Confira se criou o bucket `fotos` como **Público** no passo 1.4 |

Qualquer dúvida, o Supabase tem um chat no canto inferior direito do painel que responde em inglês mas funciona com tradutor. 💖
